# Product Sense

## 产品要解决什么问题
`aily-blockly` 的新使命不是“再做一个 Blockly 编辑器”，而是让硬件工作流像写软件一样被描述、确认、编辑与部署。核心价值是把自然语言、工作流编排与硬件配置接到一条连续链路上。

## 目标用户
- 需要快速把硬件交互想法落成工作流的开发者
- 已经熟悉开发板生态、但不想从空白工程开始的人
- 需要在 AI 辅助与手动编辑之间自由切换的教育者、创客与原型团队

## 主路径
1. 用户描述硬件场景或交互目标。
2. Tesseract Agent 生成工作流蓝图与组件建议。
3. 用户确认后进入本地 n8n 工作流编辑。
4. 桌面端继续引导配置硬件、调试串口、构建与下发。

## 产品边界
- 不是纯云端 SaaS，默认面向本地桌面运行。
- 不是彻底删除 Blockly，而是把它降为兼容路径。
- 不是泛化工作流平台，重点仍是硬件、开发板与物理交互场景。

## 一句话判断标准
如果一个改动不能让“自然语言 -> 工作流 -> 硬件配置”这条路径更短、更稳、更可解释，它就不该优先级过高。
