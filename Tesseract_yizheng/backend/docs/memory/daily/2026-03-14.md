# 2026-03-14

## New Facts

- Refactor-4 后首次端到端测试（gesture 场景）揭示了工作流生成层的核心差距：生成 5 节点 vs 目标 10 节点，缺失 50% 的节点、完全缺失 TTS/Speaker/第二人分支。
- 根因不在 harness 层（Refactor-4 已解决），而在 composition 层——缺少场景知识注入和实体驱动的拓扑构建。
- ComponentComposer 的 capability→node 是 1:1 映射，无法处理"同一能力按实体乘法展开"的模式。
- `docs/development/scene/gesture/nodes.md` 定义了完整的场景规范，但未被代码消费。

## Decisions

- 启动 Refactor-5: AI-Native Workflow Composition，核心是**上下文工程**——通过增强 LLM 看到的上下文让模型自主生成正确工作流。
- **AI-native 铁律：** 场景 ground truth 只用于验证工作流质量，绝不注入生成管线。任何场景模板硬编码方案都违反 AI-native 原则。
- 设计文档存放在 `docs/decisions/refactor-5/SCENE_DRIVEN_COMPOSITION.md`。
- P0 三件套：discovery 实体提取增强 + architect system prompt 增强（拓扑模式/notes 规范/实体乘法） + confirm 路径统一走 WorkflowArchitect（LLM 驱动）。
- P1：workflow-architect.ts（4085 行）拆分为子模块 + ground truth 自动化验证框架。
- ComponentComposer 从 confirm 主路径退化为可选 fallback。
- 三场景 ground truth 分析完成：gesture(11节点) > 90% 覆盖率、emo(16节点) > 85%、game(35节点) > 70%（允许多轮收敛）。

## Constraints

- 场景 JSON 只作为验收基准（ground truth），不作为运行时输入或 few-shot 模板。
- 实体提取由 LLM 在 discovery 阶段输出（entities + topology_hint）。
- workflow-architect.ts 拆分是渐进式，场景后处理函数保留为安全网。
- 后处理（ensureGestureIdentityFlow 等）随 prompt 增强逐步退役，不立即删除。

## Retain

- Refactor-5 是当前最重要的长期架构主题，从 Refactor-4（harness 优化）转向上下文工程。
- 三个验收基准：gesture_0315.json(12节点, N人×FACE-NET)、game_0203.json(35节点)、emo_0310.json(16节点)。
- 测试日志 `data/logs/2026-03-13T23-38-11-439+08-00.log` 记录了首次测试的完整管线输出。
- 核心认知：问题不是模型能力不足，是上下文工程的失败。LLM 有能力生成 fan-out、notes、实体分支——但没被给到足够的上下文。
- Phase 4/后续收敛的判定标准是“主文件真变薄 + 回归不退化”；现在 `workflow-architect.ts=472` 行，说明 follow-up 加了 observability 之后仍没有把安全网控制逻辑重新堆回主壳。
- 下一步“去安全网化”的关键不是直接删后处理，而是先让 prompt 约束更强、让 safety net 可观测，再逐个关闭 flags 看 ground truth 是否仍然稳定。
- E2E 优化的当前有效顺序已经明确：先修 clarification 选项退化，再补 scene-specific fragment，最后用最小 safety net 把 game/emo 的结构缺口兜住。
- SafetyNet 退役实验的方法也已经稳定：不要用 `AGENT_SCENE_SAFETY_NETS=none` 一刀切，而是按场景逐个关闭单条 safety net 做对照，看 node coverage / category / topology 的退化位置。
- 对音频修剪类 SafetyNet，测试 fixture 也必须遵守“隔离变量”原则：避免命中 generic normalizer 的命名规则，否则会把 scene-level 修剪和 node-level 清理混在一起。
- 对照测试一旦开始出现“测试文件自己像资产仓库一样膨胀”，就应该把场景 workflow fixture 抽到独立库，让矩阵文件只保留实验逻辑和断言。
