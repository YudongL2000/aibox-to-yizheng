## 节点类型命名规则

节点类型格式：`n8n-nodes-base.{节点名称}`

其中 `{节点名称}` 是将目录名首字母小写后的结果（保持驼峰命名）。

例如：
- `Webhook/` → `n8n-nodes-base.webhook`
- `ManualTrigger/` → `n8n-nodes-base.manualTrigger`
- `Set/` → `n8n-nodes-base.set`

## 节点分类

### 触发器节点 (Trigger Nodes)
- `n8n-nodes-base.manualTrigger` - 手动触发
- `n8n-nodes-base.webhook` - Webhook 触发器
- `n8n-nodes-base.schedule` / `n8n-nodes-base.cron` - 定时触发器
- `n8n-nodes-base.n8nTrigger` - n8n 触发器
- `n8n-nodes-base.workflowTrigger` - 工作流触发器
- `n8n-nodes-base.errorTrigger` - 错误触发器
- `n8n-nodes-base.sseTrigger` - SSE 触发器
- `n8n-nodes-base.localFileTrigger` - 本地文件触发器

### 核心功能节点 (Core Nodes)
- `n8n-nodes-base.set` - 设置字段
- `n8n-nodes-base.code` - 代码执行
- `n8n-nodes-base.function` - 函数
- `n8n-nodes-base.httpRequest` - HTTP 请求
- `n8n-nodes-base.if` - 条件判断
- `n8n-nodes-base.switch` - 开关
- `n8n-nodes-base.filter` - 过滤
- `n8n-nodes-base.wait` - 等待
- `n8n-nodes-base.stopAndError` - 停止并报错

### 数据处理节点 (Data Processing Nodes)
- `n8n-nodes-base.html` - HTML 处理
- `n8n-nodes-base.xml` - XML 处理
- `n8n-nodes-base.transform` - 数据转换
- `n8n-nodes-base.itemLists` - 列表处理
- `n8n-nodes-base.splitInBatches` - 分批处理
- `n8n-nodes-base.compareDatasets` - 数据集比较
- `n8n-nodes-base.renameKeys` - 重命名键
- `n8n-nodes-base.moveBinaryData` - 移动二进制数据

### 文件操作节点 (File Nodes)
- `n8n-nodes-base.files` - 文件操作
- `n8n-nodes-base.readBinaryFile` - 读取二进制文件
- `n8n-nodes-base.readBinaryFiles` - 读取多个二进制文件
- `n8n-nodes-base.readPdf` - 读取 PDF
- `n8n-nodes-base.writeBinaryFile` - 写入二进制文件
- `n8n-nodes-base.spreadsheetFile` - 电子表格文件

### 数据库节点 (Database Nodes)
- `n8n-nodes-base.postgres` - PostgreSQL
- `n8n-nodes-base.mySql` - MySQL
- `n8n-nodes-base.mongoDb` - MongoDB
- `n8n-nodes-base.redis` - Redis
- `n8n-nodes-base.snowflake` - Snowflake
- `n8n-nodes-base.oracle` - Oracle

## 注意事项

1. 节点类型名称严格遵循驼峰命名规则（camelCase）
2. 首字母小写，后续单词首字母大写
3. 特殊缩写（如 API、HTML、XML、MQTT 等）保持原样
4. 某些节点可能有多个版本（如 `Set` 节点有 v1, v2, v3.1, 3.2, 3.3, 3.4 等版本）
