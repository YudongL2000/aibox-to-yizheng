Intake Agent (需求架构师)
1. 核心职能：
  - 角色定位：语义解析与意图分类器。
  - 边界：负责厘清用户的工作流需求并进行搭建。基于现有的硬件组件的节点能力边界，通过用户的语义进行理解判定，最后生成一份符合 n8n 工作流定义的 JSON 结构化文档。
2. 输入
  - 用户原始 （如：“我想做一个见到老付就夸他的机器人”）。
3. 输出 
  - 未收敛时：引导性 （如：“听起来很有趣，这是属于人脸识别交互的范畴吗？”）。
  - 收敛后：触发锚点 #@%SCENARIO_JSON。
4. 触发机制：
  - 启动触发：用户首次在 Workspace 输入框发送消息时。
  - 结束触发：一旦判定用户需求符合预设工作流，输出 #@%SCENARIO_JSON 标识符，随后将控制权移交给 Config Agent。
System Prompt (逻辑核心)：
你是意图锁定专家。你的任务是根据用户的需求，输出符合 n8n 节点定义的结构化 JSON 文档。一旦输出 #@%SCENARIO_JSON，你的使命即刻结束。

Demo场景运行过程（Template）
1. 个性化手势交互
- 用户初始输入：
 "见到老刘竖个中指骂人，见到老付比个V打招呼"
- 部署完成后：
机器人用底盘在空间内进行转向，扫描到老刘人脸，机器人触发机械手竖中指，说一句“滚”；扫描到老付人脸，机器人触发机械手比个V，说“你长得好帅”
- 涉及到的硬件组件工作流节点：
机械手，摄像头，喇叭，底盘（全向轮）

2. 情感交互
- 用户初始输入：
 "我想做一个和我共情的机器人"
AI可以询问用户细节，比如需要用表情吗，或者你喜欢什么风格的声音？（tts）
- 部署完成后：机器人可以通过 
语音识别+视觉识别判断用户情绪， 输出不同表情及tts 语音，并用机械臂完成一些表达情绪的动作
例如，用户说自己很难过，机器人的眼睛表现出难过的emoji，然后慢慢把机械臂放下来，说“怎么了呀主人”
           用户对着机器人笑，机器人的眼睛表现出高兴，抬起机械臂，反复夹两下钳子（机械臂动作提前预制）
- 涉及到的硬件组件工作流节点：
机械臂，摄像头，喇叭，屏幕，麦克风
