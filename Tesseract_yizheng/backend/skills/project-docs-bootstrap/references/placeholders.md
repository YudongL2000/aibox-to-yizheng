# Placeholder Rules

Use placeholders to create navigable structure without pretending the docs are complete.

## General Rules

- Keep first-pass content to 3-5 lines or a short bullet list.
- Say what the page is for, who should read it, and what still needs to be filled in.
- Favor clarity over elegance; a plain sentence beats a clever title with no explanation.

## README.md

- Keep it as the docs index.
- Group links by lifecycle stage.
- Avoid dumping operational detail here; link out instead.

## Section Pages

- Start with a one-line purpose statement.
- Add 2-4 bullets describing what belongs on the page.
- If the page is not ready, say so explicitly instead of leaving empty headings.

## Existing Projects

- If the repo already contains `doc/` and `docs/`, choose one source of truth before moving content.
- Migrate current guidance first. Archive or label historical material so it does not compete with live docs.
