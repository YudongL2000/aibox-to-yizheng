## Decisions

- emo 场景后处理必须严格退回 T2 角色：只补缺和去重，不允许继续当模板注入器。
- “LLM 输出是否能自举到可用区间”用独立 fixture + ground truth 评估来衡量，而不是继续只看全 net ON 的结果。
- Phase 2 的 coverage 阈值应由“轻缺口 fixture”承载；否则测到的是 SafetyNet 的救火能力，不是 prompt 强化后的剩余退步幅度。
- CI 集成不一定要额外再造 workflow；只要质量门测试进入默认 `npm test` 路径，现有 `.github/workflows/test.yml` 就会自动承接它。
- dormant 机制必须是真正的“影子执行 + 只告警不改写”；否则它只是换皮的 enabled。
- 质量总结必须强制拆成“测试文件回放”和“真实 E2E 执行”两层；两类证据不能再混成一个总分。
- Refactor-5 当前真实验收应以 stabilized profile 为准，raw profile 只用于观察去 SafetyNet 后的裸能力，不作为交付基线。

## Constraints

- ground truth 仍然只用于评估，不进入运行时生成管线。
- 增量补全测试要避免把通用 normalizer 的裁剪行为误判成 emo flow 的行为。
- 需要保留现有“严重残缺 fixture”测试，因为它们仍然负责证明 SafetyNet 的必要性；Phase 2 新增的是另一层“强化后退步幅度”测试。
- emo 场景的质量门不能直接复用 Phase 1 的半成品结构化 fixture；质量门应使用更接近 ground truth 的输入，而 Phase 1 fixture 保留给“模板替换退场”实验。
- 候选退役网的现实证据优先级高于 fixture：先看真实日志里是否还会生成对应脏拓扑，再决定是否默认转 dormant。
- Ground Truth 总结文档不应只写“现在都绿了”，还要同时写清“当前评测覆盖了什么、没覆盖什么”，否则团队会把结构化回放误读成开放式真实输入的全面胜利。
- 当真实 E2E 与结构化矩阵出现偏差时，验收必须服从真实 E2E；测试文件只能证明实验室内收敛，不能替代运行时事实。

## Retain

- T1 safety net 最大的问题不是“硬编码难看”，而是它会切断 prompt 改进的反馈回路。
- emo 场景的正确推进顺序是：先增强 fragment，再把后处理降级为增量补全，再观察 OFF coverage。
- 验证“从模板替换降级为增量补全”时，测试必须检查非核心 helper 节点仍被保留，否则无法证明整图覆盖真的被移除了。
- 当目标是评估 prompt 增强后的 SafetyNet 剩余必要性，fixture 应从 ground truth 只抹去一个局部结构，而不是重新手写一张低质量 workflow。
- 质量门和对照矩阵应分工：quality-gate 锁最低交付线，safety-net-matrix 追踪开/关差值与退役空间。
- `dormant` 是安全网退役前最稳的中间态：先只观测真实 LLM 是否还会犯错，积累够证据再彻底 disable。
- 评估质量增进时，必须把“全网开启的当前对齐度”和“单网关闭后的退步幅度”并排写；只给一个总分，会掩盖 SafetyNet 依赖的真实形态。
- fixture / matrix 全绿不等于真实 E2E 全绿，尤其在 `game` 这类 live 漂移更大的场景里，两层结论必须分开写。
- `workflow_ready` 只能证明生成完成；`ConfigWorkflowOrchestrator: ConfigAgent initialized` 才能证明 `/api/workflow/create` 也已经跑通。
