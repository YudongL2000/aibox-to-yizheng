# Quality Score

这是一个定性评分卡，不是 CI 自动采样结果。它的用途是帮助团队判断当前文档与产品整合成熟度，而不是代替测试报告。

| 维度 | 分数 | 说明 |
| --- | --- | --- |
| 产品方向清晰度 | 4/5 | Tesseract-first、workflow-first、Blockly fallback 的边界已经明确。 |
| 架构收敛度 | 4/5 | Electron 宿主、Tesseract sidecar、本地 n8n 的职责分层清楚。 |
| 文档可导航性 | 4/5 | 文档已收敛到单一 `docs/` 根，但仍需持续维护新旧路径一致性。 |
| 发布成熟度 | 2/5 | 当前仍是 MVP，依赖兄弟目录构建产物与本地桌面环境。 |
| 可靠性自动化 | 3/5 | 已有 prepare/smoke 检查，但跨仓依赖和环境差异仍重。 |
| 安全约束显性化 | 3/5 | 路径限制、命令白名单和审计规则已文档化，仍需要实现与文档长期同构。 |

结论:
- 这是一个“方向对了、落地还在收口”的桌面工作台。
- 下一阶段最值得继续压缩的是跨仓依赖、运行时启动前提和历史兼容分支复杂度。
