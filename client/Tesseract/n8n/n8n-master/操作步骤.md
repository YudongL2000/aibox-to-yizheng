# n8n 项目启动操作步骤

## 🎯 目标
启动 n8n 开发服务器，访问自定义页面 http://localhost:8080/my-custom-page

---

## 📋 完整操作步骤

### 步骤 1：检查当前状态

打开终端，执行以下命令检查是否有服务器在运行：

```bash
# 检查端口 5678（后端）
lsof -i :5678

# 检查端口 8080（前端）
lsof -i :8080
```

**如果有进程在运行，记下 PID，需要先停止它们。**

---

### 步骤 2：停止已运行的服务器（如果有）

如果步骤 1 发现有进程在运行：

```bash
# 停止进程（替换 <PID> 为实际的进程 ID）
kill -9 <PID>

# 或者停止所有 node 进程（谨慎使用）
pkill -f "pnpm dev"
```

---

### 步骤 3：进入项目目录

```bash
cd /Users/liuxiaolin/Desktop/n8n_base/n8n-master
```

---

### 步骤 4：确认 Node 版本

```bash
node -v
```

**应该显示：`v24.13.0`**

如果不是，执行：
```bash
source ~/.bashrc
nvm use 24
node -v
```

---

### 步骤 5：启动开发服务器

执行以下命令启动完整的开发环境：

```bash
pnpm dev
```

---

### 步骤 6：等待服务器启动

**重要：需要等待两个服务器都启动完成！**

你应该看到类似以下的输出：

```
# 1. 首先会看到编译信息（很多包的编译日志）
...

# 2. 后端服务器启动成功（看到这个说明后端启动完成）
n8n ready on 0.0.0.0, port 5678

# 3. 前端开发服务器启动成功（看到这个说明前端启动完成）
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:8080/
➜  Network: http://192.168.x.x:8080/
```

**⏰ 首次启动可能需要 3-5 分钟，请耐心等待！**

---

### 步骤 7：验证服务器是否启动成功

**打开新的终端窗口**（不要关闭运行 `pnpm dev` 的终端），执行：

```bash
# 检查后端服务器
curl http://localhost:5678 | head -3

# 检查前端服务器
curl http://localhost:8080 | head -3
```

如果两个命令都返回 HTML 内容，说明启动成功。

---

### 步骤 8：访问页面

在浏览器中打开：

**http://localhost:8080/my-custom-page**

**⚠️ 注意：**
- ✅ 使用端口 **8080**（前端开发服务器）
- ❌ 不要使用端口 5678（后端服务器）

   ```bash

   pnpm dev
   ```

5. **访问页面**：http://localhost:8080/my-custom-page

## ❌ 常见错误及解决方法

### 错误 1：端口被占用

**错误信息**：`Port 5678 is already in use` 或 `Port 8080 is already in use`

**解决方法**：
```bash
# 查找占用端口的进程
lsof -i :5678
lsof -i :8080

# 停止进程（替换 <PID>）
kill -9 <PID>
```

### 错误 2：Node 版本错误

**错误信息**：`Unsupported engine: wanted: {"node":">=20.19 <= 24.x"}`

**解决方法**：
```bash
source ~/.bashrc
nvm use 24
node -v  # 确认是 v24.13.0
```

### 错误 3：依赖未安装

**错误信息**：`Cannot find module`

**解决方法**：
```bash
cd /Users/liuxiaolin/Desktop/n8n_base/n8n-master
pnpm install
```

### 错误 4：前端服务器未启动

**现象**：访问 http://localhost:5678 看到 "JavaScript enabled" 错误

**解决方法**：
- 确保运行的是 `pnpm dev`，不是 `pnpm dev:be`
- 等待前端服务器启动完成（看到 VITE 输出）
- 访问 http://localhost:8080 而不是 5678

## 💡 提示

- ⏰ **首次启动需要时间**：编译所有包可能需要 3-5 分钟
- 🔄 **热重载**：开发模式下，修改代码会自动刷新
- 🛑 **不要关闭终端**：运行 `pnpm dev` 的终端需要保持运行
- 🌐 **使用正确的端口**：开发模式访问 8080，生产模式访问 5678
